工厂模式：在创建对象时不对对客户端暴露创建逻辑，并且通过使用一个共同接口来指向新的创建对象。

意图：定义一个创建对象的接口，让其子类自己决定实例化哪个工厂类，工厂模式使其创建过程延迟到子类进行。

主要解决：接口选择问题


应用实例：需要一辆汽车，可以直接从工厂里面提货，而不用去管这辆车是怎么做出来的，以及汽车里面的具体实现。hibernate切换数据库只需要换方言和驱动就可以了。

使用场景：日志记录器：记录日志可以到本地硬盘，系统事件，远程服务器等等。2.数据库访问，用户可以选择记录日志到什么地方，需要知道使用什么数据库，以及数据库可能有的变化。3.连接器的设置，pop3 imap http 可以作为三个产品进行实现。
